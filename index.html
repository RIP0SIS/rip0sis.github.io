  • Dev Experience: `data.sql` for bootstrapped demo data; profiles for quick dev ↔ prod switch.
  • API Docs & Observability: Integrated `springdoc-openapi` (Swagger UI) and Spring Boot Actuator for metrics & health endpoints.

Spring Boot To-Do Management Application — Details
Overview

Hybrid backend that serves both a JSP-based MVC UI and a JSON REST API — same domain model reused for both interfaces, demonstrating clean separation and reusability.

Key Features
  • Comprehensive RESTful endpoints (CRUD) with ResponseEntity and correct HTTP statuses.
  • Bean Validation with @Valid and centralized @ControllerAdvice error mapping (structured JSON errorDetails payload).
  • Custom NotFoundException + centralized exception handling for consistent API consumers.
  • Security: Spring Security with BCrypt, in-memory demo users & role-based restrictions; session handling where needed.
  • Two DB profiles: H2 for quick local dev and MySQL (Docker) for production readiness — configuration hints included.
  • Bootstrapped demo data via data.sql and Docker-friendly settings for easy demos.
  • Integrated springdoc-openapi to auto-generate interactive OpenAPI/Swagger documentation for REST endpoints.
  • Enabled Spring Boot Actuator to expose operational endpoints (health, metrics) for basic observability.
  • Applied static filtering on REST responses to control exposed fields and keep API contracts clean and intentional.
